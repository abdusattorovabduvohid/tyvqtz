# Админ Панель

Full-stack админка на **Next.js 14 (App Router)** + **Mantine UI** + **Prisma/SQLite**.
Анимации — **Framer Motion**. Авторизация — JWT в httpOnly-cookie. Права — RBAC (секции × действия).

## Запуск

```bash
npm install
npm run db:push     # создать БД из схемы
npm run db:seed     # суперадмин + демо-роль
npm run dev         # http://localhost:3000
```

Тестовый вход: **admin / admin123**

## Скрипты

| Команда | Назначение |
|---|---|
| `npm run dev` | dev-сервер |
| `npm run build` / `npm start` | прод-сборка и запуск |
| `npm run db:push` | синхронизировать схему с БД |
| `npm run db:seed` | заполнить начальными данными |
| `npm run db:studio` | Prisma Studio (просмотр БД) |

## Что реализовано (ядро)

- **Логин** — анимированная страница, защита маршрутов через `middleware`.
- **Пользователи** — список (поиск, аватары) + создание/редактирование/удаление.
  Поля: логин, имя, фамилия, отчество, фото (загрузка), пароль, роль.
- **Роли** — список карточками + создание/редактирование с матрицей прав
  (секции × действия view/create/update/delete). Переключатель «Суперадмин» = все доступы.
- **RBAC** — меню и кнопки скрываются по правам; API защищён на сервере (`requirePermission`).

## Вагоны и этапы (реализовано)

- **Типы вагонов** — справочник (CRUD), название типа.
- **Этапы** — справочник этапов (номер, название, время на выполнение).
  Кнопка «Сгенерировать 23» создаёт стандартный набор этапов.
- **Вагоны** — создание (название, номер, тип). При создании вагон получает
  **снимок всех этапов** (23) по порядку.
- **Карточка вагона** — таймлайн этапов: назначение исполнителей, старт/завершение,
  **живой обратный отсчёт** (серверное время + polling каждые 5 сек), статусы
  (ожидает / в работе / просрочен / завершён). Этапы идут строго последовательно.
- **Мои этапы** — рабочий интерфейс исполнителя: назначенные ему этапы с таймерами
  и кнопками «Старт» / «Завершить».
